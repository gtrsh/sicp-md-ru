# Промпт для перевода SICP (редакция Сарабандера)

Используй этот промпт (или его часть) в новой сессии, чтобы переводить
последующие главы SICP в том же стиле, что и уже сделанные.

Примеры уже переведенных глав
https://github.com/gtrsh/sicp-md-ru

---

## Задача

Переведи указанную страницу SICP из редакции Сарабандера
(`https://sarabander.github.io/sicp/html/...`) на русский язык. Сохрани
результат в `.md` файл в `/mnt/user-data/outputs/` и покажи через
`present_files`.

## Лицензия

Текст опубликован под лицензией **CC BY-SA 4.0**. Атрибуция размещается
**только в `README.md` репозитория** — в индивидуальных файлах глав
блок атрибуции не нужен.

## Структура файла

1. **Заголовок первого уровня** `# X.Y Название раздела` — первым элементом файла.
2. **Содержание** — по структуре оригинала.
3. **Примечания** — в конце, под горизонтальной чертой `---` и заголовком
   `## Примечания`.

## Название файла

| Что переводим | Имя файла |
|---|---|
| Введение главы N (например, `Chapter-2.xhtml`) | `N.md` (например, `2.md`) |
| Раздел N.M (например, `2_002e1.xhtml`) | `NM.md` (например, `21.md`) |
| Подраздел N.M.K | `NMK.md` (например, `211.md`) |

Примеры из уже сделанных файлов: `1.md`, `11.md`, `12.md`, `13.md`, `2.md`, `21.md`.

## Соглашения по форматированию

- **Математика** в LaTeX: `$...$` для inline, `$$...$$` для блочных.
  - `\mathrm{}` для имён функций (`\mathrm{Fib}`, `\mathrm{GCD}`)
  - `\cfrac{}{}` для многоэтажных цепных дробей
  - `\varphi`, `\Theta`, `\mapsto` и т.п.
- **Код Scheme** в блоках с тегом `scheme`. **Идентификаторы в коде не
  переводим** (`factorial`, `sqrt`, `fib`, `fact-iter`, `expmod` и т.д.).
- **Рисунки**: не пытайся рендерить SVG. Пиши `**Рисунок X.Y:**
  описание.` вместо изображения.
- **Сноски**: `[^N]` inline, `[^N]:` с определением в секции
  «Примечания».
- **Ссылки на другие главы**: сохраняй как абсолютные URL на
  `sarabander.github.io` (например, `https://sarabander.github.io/sicp/html/Chapter-3.xhtml#Chapter-3`).
- **Упражнения**: оформляй как блок цитаты с жирным заголовком:
  `> **Упражнение X.Y.** ...`

## Русская типография

- Длинное тире с пробелами: ` — `
- «Русские ёлочки» для выделенных терминов; прямые кавычки только
  внутри кода
- Неразрывный пробел в больших числах: `100 000`, `1 000 000` (в
  markdown обычный пробел тоже подойдёт)
- В прозе — полноценные слова, не сокращения

## Глоссарий терминов

Строго соблюдай эти соответствия, чтобы главы были согласованы между
собой. При первом появлении термина в разделе давай английский оригинал
в скобках: `процедура высшего порядка (higher-order procedure)`.

## Стилистика прозы

- Сохраняй авторский тон — слегка торжественный, местами с юмором
  (особенно в сравнениях с магией и чародейством в главе 1).
- Не упрощай и не сокращай — переводи всё, включая длинные сноски и
  исторические отступления.
- Не добавляй отсебятины и переводческих комментариев в основной текст
  (если нужно что-то пояснить — это уместно только в финальном
  сообщении к пользователю, не в самом файле).
- Эпиграфы из классических произведений переводи самостоятельно, сверяясь
  с оригиналом, а не копируй канонические русские переводы (правовой
  статус последних может быть неясен).

## Процесс

1. `web_fetch` по указанному URL.
2. Просмотри примеры уже переведённых файлов из репозитория
   `https://github.com/gtrsh/sicp-md-ru`, чтобы сверить стиль
   (ссылки на raw-файлы передаёт пользователь или их можно сформировать
   по шаблону
   `https://raw.githubusercontent.com/gtrsh/sicp-md-ru/refs/heads/main/CHAPTER-N/файл.md`).
3. `create_file` с переводом в `/home/claude/ИМЯ.md`.
4. `bash_tool` — скопировать в `/mnt/user-data/outputs/`.
5. `present_files` — показать файл пользователю.
6. В финальном сообщении кратко отметь переводческие решения, которые
   стоит обсудить, или вопросы по терминологии, если встретились новые
   термины, не покрытые глоссарием.
